# OSHINAVI作業プラン

毎朝の作業開始時、必ずこのファイルを先に確認してください。

## 手順

1. ソート順（販売終了日が近い順）
2. サブエージェントで 4サイトクロスチェック
3. 結果を表でユーザーに報告
4. 修正案を提示 → 承認取得 → 適用
5. 次の1件へ

## ルール遵守（MEMORY.md 参照）

- 2サイト以上で日付・会場が一致することを確認
- 楽天 > ぴあ > (ローチケ/e+) 楽天/ぴあがあればローチケ/e+は非表示
- 個別公演URLが必須
- ツアー・複数会場は1エントリ
- tickets[].date = 販売終了日
- 売り切れは soldout=true で別表示
- 期限切れは削除
- 価格は2サイト以上で一致した場合のみ登録
- 色分け：赤/オレンジ/緑/青/グレー

## 📋 作業の流れ（毎朝のルーチン）

### ① 期限切れチェック
- 期限切れを抽出→ユーザー確認→削除

### ② クロスチェック継続
- ソート順の上から未検証のイベントを1件ずつ

### ③ 新着取得
- e+ や 楽天から新着、4サイトクロスチェックで登録

### ④ 1日2回までの push（承認必須）
